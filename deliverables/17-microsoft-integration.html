CHAPTER 17

HUBSPOT + MICROSOFT ENTERPRISE

10 MIN READ

Your organisation runs Microsoft's enterprise stack — Dynamics 365, Azure Active Directory, Power BI, Teams, SharePoint, and potentially Dataverse. HubSpot doesn't replace any of this. It needs to plug into it. Here's exactly how each system connects and what happens during consolidation.

THE BIG PICTURE

HubSpot handles marketing, sales pipeline, and service ticketing. Microsoft handles ERP, finance, identity, internal collaboration, and enterprise reporting. The two systems share customer data bidirectionally — when a deal closes in HubSpot, it creates a sales order in Dynamics. When a contact is updated in Dynamics, it syncs back to HubSpot.

During consolidation, every one of these connections must be preserved, tested, and re-verified. If you currently have 3-4 HubSpot portals each connected to Microsoft services, you're untangling multiple integration points and reconnecting them to one portal.

DYNAMICS 365 — THE CORE INTEGRATION

This is the most critical Microsoft connection. HubSpot's native Dynamics 365 integration supports bidirectional sync of core CRM objects:

HUBSPOT OBJECTDYNAMICS 365 OBJECTSYNC DIRECTION
CompaniesAccountsBidirectional
ContactsContacts / LeadsBidirectional
DealsOpportunitiesBidirectional
ProductsProductsBidirectional
InvoicesInvoicesBidirectional
Activities (meetings, calls, emails)Appointments, Phone Calls, EmailsBidirectional
SETUP REQUIREMENTS

Super Admin in HubSpot + Global Admin in Azure AD. You can connect up to two Dynamics 365 instances per HubSpot portal. Base integration is free on all plans. Custom field mappings require Data Hub (paid). Automated sales order creation from deal workflows requires Sales Hub Professional or Enterprise.

WHAT CHANGES DURING CONSOLIDATION

If each of your 3-4 portals currently connects to Dynamics 365 separately:

  • Before: 3-4 separate HubSpot → Dynamics sync connections. Same Dynamics account receives data from multiple portals. Potential for duplicate records in Dynamics too.
  • After: 1 HubSpot → Dynamics sync connection. Clean, deduplicated data flowing from a single source. Business Unit tags in HubSpot map to business units or divisions in Dynamics.
CRITICAL DURING MIGRATION

Disconnect Dynamics integrations from source portals before migrating data to the target portal. If you migrate while the sync is active, Dynamics will see the incoming records as new and may create duplicates on the Microsoft side. Sequence: disconnect old sync → migrate data → reconnect sync on target portal → verify.

AZURE ACTIVE DIRECTORY — IDENTITY & ACCESS

Azure AD (now Microsoft Entra ID) is how your organisation manages user identity. HubSpot connects to it for:

  • Single Sign-On (SSO) — users log into HubSpot with their Microsoft credentials. No separate HubSpot password.
  • OAuth authentication — HubSpot's Dynamics integration and MCP connections authenticate through Azure AD.
  • User provisioning — when someone joins or leaves in Azure AD, their HubSpot access can be managed centrally.

During consolidation: SSO configuration moves to the target portal. Global Admin grants HubSpot app permissions in Azure AD for the new portal. Old portal app registrations are revoked after migration is verified.

POWER BI — ENTERPRISE REPORTING

Power BI pulls data from HubSpot to build executive dashboards, cross-system reporting, and advanced analytics that combine CRM data with financial data from Dynamics.

CONNECTION METHODS

  • HubSpot API → Power BI — direct API connection, real-time data. Requires a connector (CData, Coupler.io, or custom).
  • HubSpot → Dataverse → Power BI — sync HubSpot data to Microsoft Dataverse (the database behind Dynamics), then Power BI reads from Dataverse. Most enterprise approach.
  • HubSpot Data Hub → Warehouse → Power BI — push HubSpot data to Snowflake/BigQuery via Data Hub, connect Power BI to the warehouse. Best for combining with non-Microsoft data sources.

During consolidation: Power BI reports that pulled from 3-4 separate HubSpot connections now pull from one. Every dashboard that referenced a specific portal's API key needs updating to the target portal's credentials. The upside — reports that previously showed partial data now show everything.

MICROSOFT TEAMS — COLLABORATION

The HubSpot-Teams integration sends CRM notifications directly into Teams channels — new deals, ticket updates, lead assignments. Sales and service teams stay informed without switching apps.

During consolidation: Reinstall the Teams integration on the target portal. Reconfigure which channels receive which notifications. With Business Units, you can route Product A notifications to the Product A channel and Product B to its own channel — cleaner than the current setup where each portal had its own Teams connection.

MICROSOFT DATAVERSE — THE ENTERPRISE DATA LAYER

Dataverse is the cloud database that underpins all of Dynamics 365 and the Power Platform (Power Apps, Power Automate, Power BI). If your organisation uses Dataverse as a central data hub, HubSpot data should flow into it.

ENTERPRISE ARCHITECTURE

HOW DATA FLOWS

HubSpot (Marketing + Sales + Service) ↕ bidirectional sync Dynamics 365 (ERP + Finance + Operations) ↕ native connection Microsoft Dataverse (unified data layer) ↕ read access Power BI (executive dashboards) Power Apps (custom business apps) Power Automate (workflow automation)

After consolidation, this architecture becomes cleaner — one HubSpot portal feeding one Dynamics connection feeding one Dataverse instance. No more 3-4 parallel data streams creating conflicts and duplicates in your Microsoft stack.

SHAREPOINT & ONEDRIVE — DOCUMENT MANAGEMENT

HubSpot can link to SharePoint for document storage — proposals, contracts, and attachments stored in SharePoint and referenced from HubSpot deal records. During consolidation, document links in migrated records should still resolve to the same SharePoint locations. Test a sample of document links after migration to verify.

OUTLOOK — EMAIL & CALENDAR

Individual users connect their Outlook accounts to HubSpot for email tracking and calendar sync. During consolidation:

  • Each user needs to reconnect their Outlook account to the target portal
  • Historical email tracking data migrates with the contact records
  • Calendar sync (meetings) needs re-authentication per user
  • The HubSpot Outlook add-in must be reinstalled pointing to the new portal

CONSOLIDATION CHECKLIST — MICROSOFT STACK

  • Disconnect Dynamics 365 sync from all source portals before migration
  • Reconnect Dynamics 365 to target portal — verify bidirectional sync
  • Update Azure AD app registration — revoke old portal, grant new portal
  • Reconfigure SSO to point to target portal
  • Update Power BI connections — new API credentials for target portal
  • Reinstall Teams integration on target portal — reconfigure channel routing
  • Verify Dataverse sync if applicable
  • Test SharePoint document links on migrated records
  • Each user reconnects Outlook + calendar + reinstalls add-in
  • Verify MCP OAuth through Azure AD for AI-powered access
THE PAYOFF

After consolidation, your Microsoft stack connects to one HubSpot portal instead of 3-4. Power BI dashboards show complete data. Dynamics sync is clean and deduplicated. Teams notifications are organised by Business Unit. SSO works through one app registration. The complexity drops dramatically — and so does the maintenance burden on IT.
