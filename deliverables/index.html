01
ENGAGE HUBSPOT PARTNER
FIRST POINT OF CALL

Before anything else, engage a certified HubSpot Solutions Partner. This is not optional — partners have tools and access that you don't:

  • Migration support credits — Partners can request HubSpot-funded migration assistance on your behalf
  • Enterprise pricing negotiation — Partners have direct lines to HubSpot's pricing team and can secure bundle discounts you can't get self-serve
  • Dedicated migration manager — HubSpot assigns a migration specialist to partner-led projects for direct support
  • Beta feature access — Partners get early access to migration tools, sandbox improvements, and MCP capabilities before general release
  • Portal-level diagnostics — Partners can run health checks and data quality scans that aren't available in the standard HubSpot UI

A partner de-risks the entire project. They've done this before — you haven't.

EDWARD — PROJECT LEAD HUBSPOT PARTNER
02
IT APPROVAL FOR MCP AI DISCOVERY
WEEK 1 — BEFORE ANY DATA ACCESS

Before Claude touches any data, IT must review and approve the MCP connection. This is a governance checkpoint — no AI reads your CRM until your IT team signs off on the security, scoping, and access controls.

What MCP is (for IT): Model Context Protocol is an open standard created by Anthropic and adopted across the industry. It's a secure, scoped bridge between AI tools and business systems. It does NOT copy your database, does NOT store your data externally, and does NOT require opening firewall ports. All communication runs over HTTPS with OAuth 2.0 authentication. The AI only sees what your admin explicitly grants through API scopes.

Who's already using it:

  • HubSpot — built and maintains the official HubSpot MCP Server (public beta, launched 2026)
  • Salesforce — Agentforce platform uses MCP for AI agent integration across their CRM
  • Microsoft — Copilot Studio supports MCP connections for enterprise data access
  • Google — Gemini integrations support MCP for Google Workspace and Cloud services
  • Atlassian — Jira and Confluence offer MCP servers for AI-powered project management
  • Block (Square) — uses MCP for AI-driven commerce analytics
  • Apollo, Intercom, Zendesk — all have production MCP integrations

Security controls IT reviews:

  • Scoped access — Admin sets exactly which objects Claude can read (contacts yes, financial data no). Read-only for discovery phase.
  • OAuth 2.0 authentication — standard enterprise auth. No passwords shared. Tokens are revocable at any time.
  • No data storage — Claude processes data in-session. It does not copy, cache, or retain your CRM data after the session ends.
  • Sensitive fields excluded — HubSpot's MCP server automatically blocks access to custom Sensitive Data Properties (PHI, financial, highly sensitive fields).
  • Audit trail — every MCP request is logged in HubSpot's activity log. IT can see exactly what was accessed and when.
  • Revocable any time — IT can disconnect the MCP connection instantly. One click, access gone.

What IT approves: A document listing: which portals Claude will access, which object scopes are requested (read-only), who authorised the connection, and the business justification (migration discovery). Standard IT change request process.

03
AI-POWERED DISCOVERY VIA MCP
WEEK 1

Claude connects to each portal through HubSpot's MCP server. It reads every contact, company, deal, ticket, property, workflow, and integration — across all portals — and builds a complete map of your entire data landscape. What used to take weeks of manual spreadsheet work happens in hours.

How it works: MCP (Model Context Protocol) gives Claude secure, read-only access to your HubSpot data. You ask "Show me all custom properties across Portal 1 and Portal 2" — Claude pulls both, compares them, and highlights where field names differ for the same data. No manual exports. No spreadsheets.

Data integrity check: During discovery, Claude also verifies the health of your existing data — orphaned records (contacts with no company), broken associations (deals linked to deleted contacts), empty required fields, and inconsistent lifecycle stages. Problems found here get fixed before migration, not during.

Email account safety: Claude documents every email sending domain, its authentication status (DKIM, SPF, DMARC), sending reputation, and connected mailboxes across all portals. This becomes the email integrity baseline — nothing moves until every domain is accounted for and a protection plan is in place.

EDWARD — PROJECT LEAD CLAUDE + MCP
04
AUDIT & MAP
WEEK 1-2

Claude takes the MCP discovery data and generates every deliverable for this phase automatically: the property mapping table (which fields match, which conflict), the duplicate contact count (email-matched across portals), the integration inventory, and the email domain audit.

Department head sign-off: Each department head reviews the audit for their portal. Sales VP confirms the deal data is accurate. Service Manager confirms ticket history is complete. Marketing Director confirms email lists and campaign data. Nobody's data moves until the person who owns it confirms it's correct.

Email integrity checkpoint: Every connected email account, shared inbox, and sending domain is documented. The partner confirms that no email routing, forwarding rules, or connected mailboxes will break during migration. Personal email connections (Gmail/Outlook sync) are flagged for re-authentication planning.

CHOOSE TARGET PORTAL
WEEK 2

Partner presents a recommendation with scored decision matrix. Leadership decides: keep the biggest portal, keep the best-configured, or start fresh. The MCP audit data feeds directly into the scoring.

06
SANDBOX TESTING
WEEK 3-4

Run a full test migration in HubSpot's sandbox. Zero production risk. The partner runs the Datawarehouse.io migrator tool. After migration, Claude connects to the sandbox via MCP and cross-checks: "Do the record counts match the source? Are associations intact? Did all properties come across?"

Department head walkthrough: Each department head gets sandbox access to verify their data personally. Sales VP spot-checks 20 deals — are amounts, stages, and contact associations correct? Service Manager checks ticket history — are notes and attachments there? Marketing Director verifies email lists and subscription preferences. This is the last gate before production migration.

Email sending test: Partner connects each brand's email domain to the sandbox and sends test emails. Team verifies: correct from address, correct branding, DKIM authentication passes, emails land in inbox not spam. Email integrity is confirmed before any production data moves.

EDWARD — PROJECT LEAD PARTNER EXECUTES CLAUDE VALIDATES HEADS VERIFY
07
PRE-MIGRATION PREP
WEEK 4-5

Configure Business Units, rebuild reports and dashboards, install deduplication tools. Set a freeze date — all teams stop creating records in old portals from this point.

Email domain protection: Every brand's email domain is connected to the target portal before being removed from old portals. Both old and new portals run in parallel with the same authenticated domains — this is safe and expected. DKIM, SPF, and DMARC records are verified green in the new portal before any sending switches over. No domain is left unauthenticated at any point.

Full data backup: Every portal is exported as CSV before the freeze — contacts, companies, deals, tickets, custom objects, engagements. These backups are stored outside HubSpot (shared drive or cloud storage) as a "break glass" recovery option. Old portals remain read-only for 90 days after migration. Nothing is deleted. Your historical data is safe in three places: the target portal, the CSV exports, and the original portals (read-only).

EDWARD — PROJECT LEAD PARTNER CONFIGURES TEAMS FREEZE
08
EXECUTE MIGRATION
WEEK 5-6

Migrate data portal by portal. Verify record counts after each. Run deduplication (email-match contacts, domain-match companies). Assign Business Unit tags. Reconnect integrations. Invite users and assign seats.

EDWARD — PROJECT LEAD PARTNER MIGRATES CLAUDE VERIFIES
09
EMAIL WARM-UP & GO-LIVE
WEEK 6-8

Gradually ramp email volume over 4 weeks — start with your most engaged contacts (opened/clicked in last 30 days), expand weekly to 60-day, 90-day, then full list. This protects your domain reputation with email providers like Gmail and Outlook.

Email integrity maintained: All email sending history, subscription preferences, and unsubscribe lists carry over. If someone unsubscribed from Product A emails in the old portal, they stay unsubscribed in the new portal. DMARC/DKIM/SPF authentication is verified daily during warm-up. If any bounce rate exceeds 5% or spam complaints exceed 0.3%, sending pauses immediately until diagnosed.

Old portals as fallback: Old portals stay active in read-only mode throughout warm-up. If email deliverability degrades, the partner can temporarily re-route sending through the old portal while the issue is resolved. No emails are lost.

EDWARD — PROJECT LEAD PARTNER MONITORS MARKETING SENDS
10
CONNECT UNIFIED MCP
WEEK 8-9

With all data in one portal, replace 3-4 separate MCP connections with a single one. Claude now sees your entire customer base through one lens. Sales asks "Which Product A customers haven't tried Product B?" Marketing asks "Open rate by brand this quarter?" Service asks "Full ticket history for Company X across all products?" One connection, one answer, complete data.

Each team gets hands-on training in the new unified portal. Sales learns cross-sell views. Service learns unified ticket history. Marketing learns Business Unit campaigns. Leadership gets their new dashboards. One migration champion per team for ongoing support.
